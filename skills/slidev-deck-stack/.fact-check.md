### features 補強
- **global layers** — https://sli.dev/features/global-layers
  專案根目錄 `global-top.vue` / `global-bottom.vue` / `slide-top.vue` / `slide-bottom.vue` /
  `custom-nav-controls.vue` 自動掛載；可用 `$nav`(currentPage/total/currentLayout/next)；
  ⚠ 依賴導覽狀態時匯出要 `--per-slide`。
- **slide-scope style** — https://sli.dev/features/slide-scope-style
  Markdown 內 `<style>` **永遠 scoped**（不必加 `scoped`）；子選擇器 `.a > .b` 不可用；
  支援巢狀 CSS 與 `--uno:` 指令。
- **icons** — https://sli.dev/features/icons
  Iconify `{collection}-{icon}`；裝 `@iconify-json/<collection>`；動態名需 uno `safelist`。
- **zoom slide** — https://sli.dev/features/zoom-slide
  per-slide frontmatter `zoom: 0.8`，只影響該頁。
- **draggable elements** — https://sli.dev/features/draggable
  位置格式 `Left,Top,Width,Height,Rotate`；指令 `v-drag="[...]"`、元件 `<v-drag pos="...">`、
  具名 `dragPos:` frontmatter + `pos="name"`、`<v-drag-arrow>`；不給值會自動產生初始位置、
  雙擊可編輯；Height 給 `NaN`(指令)/`_`(元件) 自動高。
- **code groups** — https://sli.dev/features/code-groups
  `::code-group … ::` 容器、`[label]` 頁籤、`~i-xxx~` 自訂圖示；**需 headmatter `comark: true`**。
- **line numbers** — https://sli.dev/features/line-numbers（佐證：https://sli.dev/guide/syntax）
  全域 headmatter `lineNumbers: true`；逐塊 `{lines:true}`；起始行 `{startLine:5}`，可合併。
- **frontmatter 合併（src 匯入）** — https://sli.dev/features/...（features 索引）
  被 `src:` 引入頁與主檔該頁 frontmatter 合併，**同鍵主檔(entry)優先**。
- **block frontmatter** — https://sli.dev/features/block-frontmatter
  某頁可用 fenced ```yaml 區塊當 frontmatter（編輯器高亮/格式化友善）；
  **headmatter（整檔第一個 frontmatter）不能用此寫法**，只能標準 `---`。

### 重要修正（查證推翻原假設）
1. **無單頁 `rotate:` frontmatter**：Slidev 沒有獨立的單頁旋轉功能。`Rotate` 僅是 draggable
   位置的第五個值（`Left,Top,Width,Height,Rotate`）。原規劃「rotate 當 feature」已撤除，
   旋轉改在 animation.md draggable 一節說明。
2. **`comark` 與 `mdc` 是兩個獨立旗標**：`mdc: true` 開 MDC（行內 class/屬性）；
   **code groups 需另開 `comark: true`**。skill 既有 `mdc: true` 文件正確，code groups 不沿用。
3. draggable 官方頁網址是 `/features/draggable`（非 `/draggable-elements`）。
